## 1) ภาพรวมผลิตภัณฑ์

**FinFlow** = เว็บแอป (PWA) รวบรวม + วิเคราะห์การเงินส่วนบุคคล

- **กลุ่มเป้าหมาย:** คนไทยที่มีหลายกระเป๋าเงิน (KBank, Make by KBank, TrueMoney) + กรรมการการแข่งขัน
- **คุณค่าหลัก:** ดึง statement/สลิปจาก Gmail → รวมเป็นรูปแบบเดียว → วิเคราะห์ด้วยเอนจินสถิติที่เขียนเอง (Financial Health Score, Sankey, ไทม์ไลน์+พยากรณ์, ตรวจจับผิดปกติ, แนวโน้มหมวด, กันซ้ำ/จับคู่โอนข้ามกระเป๋า, ผู้ช่วย AI, ภาษี)
- **ภาษา UI:** ไทยทั้งหมด · สกุลเงินบาท (`12,345 ฿`, ไม่มีทศนิยม) · ปี พ.ศ.
- **จุดขายต่อกรรมการ (สำคัญต่อ UX):** คะแนนคณิต/สถิติถ่วงน้ำหนักมากสุด → ทุกตัวเลขต้อง **"กดดูวิธีคิด + สูตร + แหล่งอ้างอิง"** ได้ ดีไซน์ต้องรองรับ "explain panel" โดยไม่รก
- **โหมดเดโม-first:** ใช้ได้ทันทีโดยไม่ต้องล็อกอิน (มีข้อมูลตัวอย่าง 6 เดือน) — ดีไซน์ต้องสวยตั้งแต่ first load โดยไม่ต้อง setup
- **เทคโนโลยี:** React + Vite (PWA) ฝั่งหน้าบ้าน · Express + SQLite ฝั่งหลังบ้าน · เอนจินสถิติ TypeScript ใช้ร่วมกันสองฝั่ง (คำนวณสดในเบราว์เซอร์ได้ เช่น What-if, วางแผนภาษี)

## 3) องค์ประกอบกลาง (Global Shell) — มีผลทุกหน้า

### 3.1 Sidebar (กว้าง 248px, sticky เต็มจอ)
ลำดับจากบนลงล่าง:
1. **Brand** — โลโก้ (38px, radius 11px) + "FinFlow" + tagline "เส้นทางการเงินส่วนบุคคล"
2. **Nav links** 10 อัน — active = พื้นสี brand teal `#0f766e` ตัวอักษรขาว; hover = พื้นเทาอ่อน
3. **Sidebar foot** (ดันลงล่างด้วย `margin-top:auto`):
   - **Badge sync** (โผล่เฉพาะตอนกำลังดึง/เพิ่งดึงสลิป): `🔄 กำลังดึงสลิปล่าสุด…` / `✅ อัปเดตสลิป +N`
   - **PeriodPicker** (ดู 3.2)
   - **Data scope toggle** 3 ปุ่ม: `ทั้งหมด · 🟢 จริง · 🧪 เดโม`
   - **ปุ่มสลับธีม** 🌙 โหมดมืด / ☀️ โหมดสว่าง
   - ข้อความ footer "การแข่งขัน AI & คณิตศาสตร์ ครั้งที่ 11"

### 3.2 PeriodPicker (ตัวเลือกช่วงเวลา — มีผลทุกหน้า)
ปุ่มสรุป `📅 {label}` กดแล้วขยายเป็น panel มี 3 วิธีเลือก:
- **Presets:** เดือนนี้ / 3 เดือน / 6 เดือน / 12 เดือน / ปีนี้ / ทั้งหมด
- **Month stepper:** `◀ {เดือน พ.ศ.} ▶` เดินทีละเดือน
- **Custom range:** date from–to + ปุ่ม "ใช้ช่วงนี้"
- ค่าถูกจำใน localStorage; แนบเป็น query `?from=&to=` ให้ทุก API

### 3.3 Data Scope toggle
แยกข้อมูล **จริง (Gmail/อัปโหลด)** ออกจาก **เดโม (ตัวอย่าง)** — แนบ `?data=real|demo` ทุกหน้า เพื่อให้สาธิตได้โดยไม่ปนข้อมูลจริง (ค่า default = ทั้งหมด)

### 3.4 Auto-sync เบื้องหลัง
เข้าเว็บแล้วถ้าเชื่อม Gmail ไว้และไม่ได้ sync ใน 3 นาที → ดึงสลิปล่าสุดอัตโนมัติ แสดง badge สถานะมุมบน/ใน sidebar (ไม่บล็อก UI)

### 3.5 Mobile (`≤900px`)
- Sidebar เปลี่ยนเป็น **drawer สไลด์จากซ้าย** + scrim ดำโปร่ง (`rgba(0,0,0,0.4)`)
- โผล่ **topbar**: ปุ่ม ☰ + "FinFlow" + สถานะ sync
- ทุก grid (cols-2/3/4/5) ยุบเป็น 1 คอลัมน์ (cols-4/5 → 2 คอลัมน์)

## 5) รายละเอียดแต่ละหน้า

### 5.1 📊 ภาพรวม (Dashboard) — หน้าแรก
หน้าที่หนาแน่นสุด ลำดับ section:
1. **แถวสถิติ 5 ใบ (cols-5):** รายรับรวม · รายจ่ายรวม (ไม่รวมโอนระหว่างกระเป๋า) · กันเข้าออม · ออมสุทธิสะสม · อัตราการออม (เทียบเป้า 20%+)
2. **คะแนนสุขภาพ (ซ้าย) + ข้อสังเกต/คำแนะนำ AI (ขวา)** (cols-2):
   - **HealthScoreGauge:** เกจครึ่งวงกลม 0–100 + เกรด (เปราะบาง/ประคองตัว/สุขภาพดี) + toggle โปรไฟล์ `นักเรียน/ผู้ใหญ่` + 4 เสา (ใช้จ่าย·ออม·กู้ยืม·วางแผน) แบบ accordion → กดเสาเห็น metric ย่อย + **สูตร (mono) + คำอธิบาย + 📚 แหล่งอ้างอิง** + แท่ง progress + badge "ประมาณ" สำหรับเสาที่เดาจากธุรกรรม
   - **InsightFeed (ข้อสังเกตอัตโนมัติ):** รายการ ไอคอน + หัวข้อ + badge ระดับ (ดี/ระวัง/เตือน/ข้อมูล) + คำอธิบาย
   - **AdviceCard (คำแนะนำจาก AI):** ปุ่ม "✨ ขอคำแนะนำ" → ข้อความ + ที่มา (Groq/Gemini/rule-based)
3. **การ์ด pipeline (ระบบรวม+กันซ้ำ):** badge แสดง 📥 รับเข้า / ✅ คงเหลือ / 🔁 กันซ้ำ / ↔️ จับคู่โอน
4. **(cols-2):** กราฟแท่งรายรับ–รายจ่ายรายเดือน · การ์ด "แยกตามบัญชี" (💰 ยอดเงินรวมทุกบัญชี + ต่อบัญชี: ยอดคงเหลือ + เข้า/ออก + จำนวนรายการ)
5. **รายจ่ายตามหมวด:** list (ไอคอน + ชื่อ + บาท + % + progress สีตามหมวด)
6. **WhatIfSimulator (🔮 จำลองการออม):** เลือกหมวด + slider "ลด X%" → คะแนนใหม่ + delta (+/−) + เงินประหยัดรวม (คำนวณสดในเบราว์เซอร์)

> **โจทย์ดีไซน์:** หน้านี้แน่นมาก ต้องจัดลำดับสายตา (คะแนน + อัตราออม = พระเอก) และทำ explain panel ให้ดูเป็นมือโปรสำหรับกรรมการ

### 5.2 📒 รายการ (Transactions)
- **Header actions:** ⬇️ CSV · ⚙️ กฎร้านค้า (N) · ＋ เพิ่มรายการ
- **Toast** เมื่อแก้หมวด/กฎ: "✅ … — Sankey / แนวโน้มหมวด / คะแนนสุขภาพ อัปเดตตามแล้ว"
- **Filter bar:** 🔍 ค้นหาร้าน/บัญชี · เลือกเดือน · เลือกกระเป๋า · เลือกหมวด + สรุปฝั่งขวา (จำนวน · เข้า · ออก)
- **ตาราง** (แสดง 500 แถวแรก, scroll แนวนอนบนจอแคบ):
  - วันที่ + เวลา
  - ผู้รับ/ร้าน + เลขบัญชี (badge "โอนระหว่างกระเป๋า" ถ้าเป็น transfer)
  - **หมวด** = dropdown แก้ inline ทันที (optimistic update)
  - กระเป๋า (ชื่อเล่นบัญชี)
  - จำนวนเงิน (+เขียว/−แดง)
  - ปุ่ม "ตั้งกฎ" (เฉพาะรายการที่ไม่ใช่ transfer)
- **AddForm (เพิ่มรายการเอง):** วันที่ · ทิศทาง · ผู้รับ · จำนวน · หมวด · แหล่ง + validation
- **RuleModal (ตั้งกฎร้านค้า):** จับคู่ด้วย 🏦 เลขบัญชี / 🔤 ชื่อผู้รับ + ตั้งชื่อร้าน (alias) + หมวด → ใช้กับทุกรายการที่ตรงกัน + แจ้งจำนวนที่ถูกปรับ
- **Panel กฎร้านค้า:** list กฎ (badge ประเภท + ค่า + ชื่อ + หมวด) + ปุ่มลบ (คำนวณใหม่ N รายการ)

### 5.3 🌊 เส้นทางเงิน (Flow / Sankey)
- เลือกเดือน (`<input type=month>`) หรือ "ทุกเดือน"
- **การ์ดตรวจสมดุล (Flow Conservation):** badge `✅ สมดุล / ⚠️ ไม่สมดุล` + คลาดเคลื่อนสูงสุด (เงินเข้า=ออกทุกโนดกลาง = ยืนยันความถูกต้องข้อมูล)
- **SankeyChart:** รายรับ → เงินรวม → หมวดรายจ่าย/ออม ข้ามทุกกระเป๋า (สีตามหมวด)
- **Empty state** เมื่อไม่มีข้อมูลในเดือนที่เลือก

### 5.4 📈 ไทม์ไลน์ (Timeline)
Segmented control: **รายวัน / รายเดือน / รายปี**
- **รายวัน:** กราฟแท่งรายจ่าย + เส้น moving average 7 วัน (ComposedChart)
- **รายเดือน:** แท่งรายรับ/รายจ่าย + เส้นออมสุทธิ
- **รายปี:** Area เงินออมสุทธิ + การ์ด **พยากรณ์ Linear Regression** (least-squares): +1/+2/+3 เดือน + slope/เดือน + R²

### 5.5 🔮 คาดการณ์ (Forecast)
- **Card 1 — รายหมวด:** subtitle อธิบายโมเดล (Decomposition: บิลประจำ + Damped trend φ=0.8 [+ seasonal] · 80% PI) + segmented เลือก horizon (เดือน +1/+2/+3) → grid การ์ดแต่ละหมวด:
  - ชื่อหมวด + **FlagBadge "% เกินงบ"** (ส้ม/แดง ถ้า ≥40%)
  - **stacked bar:** ประจำ (สีเข้ม) + ผันแปร (สีอ่อน)
  - ยอดคาด + ช่วง low–high (80% PI) + งบ + progress vs งบ (เขียว/ส้ม/แดง)
  - บรรทัด "ประจำ X · ผันแปร Y"
- **Card 2 — ยอดรวมทุกหมวด 3 เดือน:** 3 การ์ด (mid + ต่ำสุด/สูงสุด), อธิบาย σ_total = √(Σ σ_c²)
- **Card 3 — Backtest:** MAPE โมเดล / MAPE baseline / **Skill Score** (เขียว=แม่นกว่า naive) + `<details>` **ตารางสูตรทั้ง 8 สำหรับกรรมการ** (Decomposition, บิลประจำ, Damped trend, Level EWMA, Prediction Interval, รวมทุกหมวด, โอกาสเกินงบ, Skill Score)

### 5.6 🚨 รายจ่ายผิดปกติ (Anomalies)
- **3 stat (cols-3):** ค่าเฉลี่ยรายจ่าย/วัน · SD · เกณฑ์เตือน (Q3+3·IQR)
- **list การ์ด outlier:** 🚨 + วันที่ (label ไทย) + ยอด (แดง) + badge `z=…` + ตารางรายการในวันนั้น (5 รายการแรก: ผู้รับ/หมวด/จำนวน)
- **Empty:** "ไม่พบวันที่ใช้จ่ายผิดปกติ — เยี่ยมมาก!"
- เกณฑ์: z>2 (มีนัยสำคัญ) + IQR far-out (k=3) เพราะรายจ่ายรายวันเบ้ขวา

### 5.7 🎯 งบประมาณ (Budgets)
- ผูกกับเดือนปลายช่วงของ PeriodPicker (ถ้า "ทั้งหมด" → เดือนล่าสุด)
- บรรทัดสรุป: เดือน + จำนวนงบที่ตั้งไว้
- **grid การ์ดงบ (cols-2):** ไอคอน+หมวด + badge สถานะ (อยู่ในงบ/ใกล้เต็มงบ/เกินงบ — เขียว/ส้ม/แดง) + ใช้ไป/งบ + progress + เหลือ/เกิน + % + ปุ่มแก้ไข (inline → input + ✓)
- **การ์ดเพิ่มงบใหม่:** เลือกหมวด (เฉพาะที่ยังไม่มีงบ) + วงเงิน/เดือน + ปุ่มเพิ่ม

### 5.8 🧾 ภาษี (Tax) — ปีภาษี 2567
- **Banner เตือน:** "⚠️ ประมาณการ ไม่ใช่คำปรึกษาทางกฎหมาย" (พื้นส้มอ่อน)
- Banner ประมาณการทั้งปี (×N) ถ้าเปิด annualize
- **(cols-2):**
  - **ผลการคำนวณภาษี:** เงินได้พึงประเมิน → หักค่าใช้จ่าย → หักลดหย่อนรวม → เงินได้สุทธิ → ภาษีขั้นบันได → (ภาษีขั้นต่ำ 0.5% ถ้ามี) → ก่อนเครดิต → หัก ณ ที่จ่าย → **ต้องจ่ายเพิ่ม/ขอคืน** + อัตราเฉลี่ย/ขั้นสุดท้าย + `<details>` ค่าลดหย่อนที่ใช้พร้อมมาตรา (ref)
  - **การยื่นภาษี:** ต้องยื่น (ภ.ง.ด.) + กำหนดกระดาษ/ออนไลน์ + ขอคืนได้ · **VAT แฝง** (7/107 เชิงข้อมูล)
- **ฟอร์มเงินได้ (ทั้งปี):** เพิ่ม/ลบรายได้แยกประเภท ม.40(1)–(8) + จำนวน + หัก ณ ที่จ่าย + (เงินปันผลสำหรับ 40(4)) + checkbox สมรส / ประมาณการทั้งปี + โหมดเงินปันผล (Final 10% / รวม+เครดิต)
- **ฟอร์มค่าลดหย่อน (cols-2):** 14 ช่อง — คู่สมรส · บุตร · บิดามารดา · ประกันสังคม · ประกันชีวิต · ประกันสุขภาพ · SSF · RMF · Thai ESG · ประกันบำนาญ · กองทุนสำรองเลี้ยงชีพ · ดอกเบี้ยกู้บ้าน · บริจาคทั่วไป · Easy E-Receipt + ปุ่ม "💾 บันทึก & คำนวณใหม่"
- **TaxPlanner (🎯 วางแผนประหยัดภาษี):** เลือกช่องลดหย่อน + slider จำนวน → ภาษีใหม่ + ประหยัดได้ (สด) + list คำแนะนำ "ซื้อเพิ่มได้ X → ประหยัด ~Y" (ถ้าอัตราขั้นสุดท้าย 0% แสดงข้อความว่าไม่ถึงเกณฑ์)
- **TaxAdviceCard:** ปุ่ม "✨ ขอคำแนะนำภาษี" → AI สรุปแผนประหยัด

## 6) สถานะ & รายละเอียดที่ดีไซน์ต้องเตรียมครบ

- **ทุกหน้าที่ดึง API ต้องมี 4 สถานะ:** Loading (skeleton shimmer) → Error (⚠️ + ปุ่มลองใหม่) → Empty (📭 + ข้อความ) → Data
- **โหมดมืด:** ทุกองค์ประกอบต้องมีคู่สี dark (กราฟใช้ CSS var ผ่าน recharts)
- **Inline edit + feedback ทันที:** หมวดในตาราง, วงเงินงบ, ชื่อ/ประเภทบัญชี — ใช้ toast/optimistic
- **Explain panels (สำหรับกรรมการ):** สูตร + แหล่งอ้างอิง ในคะแนนสุขภาพ / Forecast / ภาษี — discoverable แต่ collapsible (ไม่รก)
- **ตัวเลขเงิน:** เขียว=ดี/เข้า, แดง=จ่าย/ต้องจ่าย; แกนกราฟย่อ K/M; เลขบัญชี/สูตรใช้ฟอนต์ mono
- **Responsive:** ทดสอบ Dashboard (cols-5) และตาราง Transactions (scroll แนวนอน) บนมือถือ; sidebar เป็น drawer
- **Modal/Overlay:** พื้นหลังดำโปร่ง กดนอกพื้นที่ปิด, การ์ดกลางจอ max 460px
