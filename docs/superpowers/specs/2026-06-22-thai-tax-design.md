# FinFlow — ฟีเจอร์จัดการภาษีบุคคลธรรมดา (ตามกฎหมายไทย) + คำแนะนำภาษี

> Design doc · 2026-06-22 · สถานะ: อนุมัติดีไซน์แล้ว (รอรีวิว spec ก่อนทำแผน implementation)
> ปีภาษีอ้างอิง: **2567** (ยื่นต้นปี 2568)

---

## 1. เป้าหมาย & ขอบเขต

เพิ่มฟีเจอร์ภาษีสำหรับ **บุคคลธรรมดา (เน้นวัยทำงาน/ผู้ใหญ่)** ใน FinFlow ที่ **คำนวณภาษีตามกฎหมายไทยอย่างเคร่งครัด** และให้คำแนะนำเชิงรุก ทุกตัวเลขคำนวณด้วยโค้ดเราเอง (สูตรอธิบายได้) — AI ใช้เรียบเรียงเท่านั้น

**ครอบคลุม (โครงสร้างตามกฎหมายจริง — ไม่ใช่ 4 ระบบแยก):**
- **PIT (ภาษีเงินได้บุคคลธรรมดา)** = เครื่องคำนวณหลัก
- **WHT (หัก ณ ที่จ่าย)** = เครดิตภาษีที่หักไว้แล้ว นำมาหักท้าย → ต้องจ่ายเพิ่ม/ขอคืน
- **เงินได้จากการลงทุน** = รวมใน PIT (คริปโต/หุ้นนอก = เงินได้พึงประเมิน; หุ้นในตลาด SET ผ่านโบรกฯ = ยกเว้น)
- **VAT 7%** = แสดงเชิงข้อมูลเท่านั้น (บุคคลธรรมดาทั่วไปไม่ใช่ผู้เสีย VAT)

**ไม่ทำ (out of scope):** การยื่นแบบจริงกับกรมสรรพากร, ภาษีนิติบุคคล, ภาษีธุรกิจเฉพาะ, การคำนวณ VAT แบบผู้ประกอบการจดทะเบียน

**ข้อกำหนดความถูกต้อง:**
- ค่าคงที่/อัตรา/เพดานทั้งหมดอยู่ใน **ruleset เวอร์ชันเดียว** ต่อปีภาษี (แก้ปีภาษีได้ที่เดียว โดยไม่แตะสูตร)
- ทุกผลลัพธ์โชว์ที่มา/มาตรากฎหมายในแผงอธิบาย (สไตล์เดียวกับคะแนนสุขภาพการเงิน)
- ติดป้าย **"ประมาณการ ไม่ใช่คำปรึกษาภาษีตามกฎหมาย"** ทุกจุด

## 2. สถาปัตยกรรม (แนวทาง A) & โมเดลข้อมูล

เครื่องคำนวณเป็น **pure-TS ใน `shared/`** (unit-test ด้วย vitest) ใช้ซ้ำได้ทั้ง server (คำนวณ) และ client (วางแผนสด) เก็บโปรไฟล์ภาษีใน `meta` — ตรงแพตเทิร์นทั้งโปรเจกต์ (demo-first, AI rephrase-only)

### โครงสร้างไฟล์
- `shared/src/tax/types.ts` — `TaxProfile`, `IncomeItem`, `IncomeType`, `Deductions`, `TaxResult`, `SavingSuggestion`
- `shared/src/tax/rules2567.ts` — อัตรา/ค่าลดหย่อน/เพดานปีภาษี 2567 (ออบเจกต์ข้อมูลล้วน)
- `shared/src/tax/engine.ts` — `computeTax(profile, rules)` + `suggestSavings(profile, rules, result)` + `filingInfo(profile, rules)`
- `shared/src/tax/detect.ts` — `detectIncome(txns)` เดาเงินได้ตามประเภทจากธุรกรรม
- `shared/src/tax/engine.test.ts`, `shared/src/tax/detect.test.ts`
- `shared/src/tax/index.ts` (+ re-export จาก `shared/src/index.ts`)
- `server/src/routes/tax.ts` — GET ผลภาษี (auto-detect + profile), PUT บันทึก `TaxProfile`
- `server/src/db.ts` — `getTaxProfile()/setTaxProfile()` (เก็บใน `meta.taxProfile` เป็น JSON)
- `client/src/pages/Tax.tsx` + route `/tax` ใน `App.tsx` + เมนู "🧾 ภาษี" ใน `Layout.tsx`
- `client/src/lib/types.ts` — re-export ชนิดภาษี + เพิ่ม `TaxOverview` response type

### โมเดลข้อมูล (สรุป)
```ts
type IncomeType = '40(1)' | '40(2)' | '40(3)' | '40(4)' | '40(5)' | '40(6)' | '40(7)' | '40(8)';

interface IncomeItem {
  type: IncomeType;
  amount: number;          // เงินได้ทั้งปี (บาท)
  withholding?: number;    // WHT ที่ถูกหักไว้ (บาท)
  source: 'detected' | 'user'; // มาจากการเดา หรือผู้ใช้กรอก/แก้
  note?: string;
}

interface Deductions {
  // ครอบครัว
  spouse: boolean; children: number; childrenSecondChildPlus: number;
  parents: number; disabled: number; maternity: number;
  // ประกัน + ปกส.
  socialSecurity: number; lifeInsurance: number; healthInsurance: number;
  parentHealthInsurance: number; pensionInsurance: number;
  // กองทุน
  providentFund: number; rmf: number; ssf: number; thaiEsg: number; nsf: number;
  // อื่นๆ
  homeLoanInterest: number; donationGeneral: number; donationEducation: number;
  easyEReceipt: number;
}

interface TaxProfile {
  taxYear: number;            // เช่น 2567
  married: boolean;
  income: IncomeItem[];       // เดา + ผู้ใช้แก้
  deductions: Deductions;     // ผู้ใช้กรอก
  dividendMode: 'final' | 'include'; // ปันผล: final 10% (ดีฟอลต์) หรือ นำมารวม+เครดิต
  annualize: boolean;         // ประมาณการทั้งปีถ้าข้อมูล < 12 เดือน
}

interface TaxResult {
  taxYear: number;
  grossByType: Record<IncomeType, number>;
  expenseDeduction: number;   // หักค่าใช้จ่ายตามประเภท
  totalAllowances: number;    // ค่าลดหย่อนรวม (หลัง cap)
  netIncome: number;          // เงินได้สุทธิ
  progressiveTax: number;     // ภาษีวิธีขั้นบันได
  minimumTax: number;         // ภาษีวิธี 0.5% (ถ้าเข้าเงื่อนไข)
  taxBeforeCredit: number;    // max(progressive, minimum) ตามกฎ
  withholdingCredit: number;  // WHT รวม
  taxDue: number;             // ต้องจ่ายเพิ่ม (>0) หรือ ขอคืน (<0)
  effectiveRate: number; marginalRate: number;
  brackets: { from: number; to: number; rate: number; taxable: number; tax: number }[];
  allowanceBreakdown: { id: string; label: string; used: number; cap: number; ref: string }[];
}
```

### Data flow
ธุรกรรม → `detectIncome()` เดาเงินได้ → merge กับ `TaxProfile` (ผู้ใช้ยืนยัน/แก้) → `computeTax()` → `TaxResult` + `suggestSavings()` + `filingInfo()` → หน้า Tax + คำแนะนำ + แชท

### การผูกกับโปรไฟล์ "ผู้ใหญ่"
เมนู/หน้า ภาษี โชว์-เน้นเมื่อ `scoreProfile = adult` (ตามที่ขอ "สำหรับผู้ใหญ่") แต่ยังเข้าถึงได้

## 3. สูตรภาษี & กฎหมาย (encode ใน `rules2567.ts`)

### 3.1 หักค่าใช้จ่ายตามประเภทเงินได้ (ม.40)
| ประเภท | ค่าใช้จ่าย |
|---|---|
| 40(1)+(2) เงินเดือน/รับจ้าง | 50% **รวมกันไม่เกิน 100,000** |
| 40(3) ลิขสิทธิ์/กู๊ดวิลล์ | 50% ไม่เกิน 100,000 หรือตามจริง |
| 40(4) ดอกเบี้ย/ปันผล | **หักไม่ได้** |
| 40(5) ค่าเช่า | เหมาตามประเภท (บ้าน/ที่ดิน 30%) หรือตามจริง |
| 40(6) วิชาชีพอิสระ | 30% (ทั่วไป) / 60% (โรคศิลปะ) หรือตามจริง |
| 40(7) รับเหมา | 60% หรือตามจริง |
| 40(8) ธุรกิจอื่น | 60% หรือตามจริง |

MVP ใช้ค่าเหมา (standard) เป็นดีฟอลต์ต่อประเภท; ช่อง "ตามจริง" เป็น enhancement ภายหลัง

### 3.2 ค่าลดหย่อน + เพดาน + เพดานรวม
- ส่วนตัว 60,000 · คู่สมรส (ไม่มีเงินได้) 60,000 · บุตร 30,000/คน (คนที่ 2 เกิด ≥2561 = 60,000) · บิดามารดา 30,000/คน (≤4 คน, อายุ ≥60, เงินได้ ≤30,000) · ผู้พิการ 60,000/คน · ฝากครรภ์+คลอดบุตร ≤60,000/ท้อง
- ประกันสังคม (ตามจ่ายจริง, เพดาน ~9,000) · ประกันชีวิต+สุขภาพตนเอง ≤100,000 (ส่วนสุขภาพ ≤25,000) · ประกันสุขภาพบิดามารดา ≤15,000
- **กลุ่มเกษียณ รวมกัน ≤ 500,000:** PVD/กบข./ครูเอกชน (≤15% เงินได้) + RMF (≤30%) + SSF (≤30%, ≤200,000) + ประกันบำนาญ (≤15%, ≤200,000) + กอช. (≤30,000)
- **ThaiESG แยกต่างหาก:** ≤30% เงินได้, ≤300,000
- ดอกเบี้ยกู้ซื้อบ้าน ≤100,000
- บริจาคการศึกษา/กีฬา/รพ.รัฐ = ×2 · บริจาคทั่วไป = ตามจริง → **รวมบริจาคทั้งหมดไม่เกิน 10% ของเงินได้หลังหักค่าลดหย่อนอื่น**
- Easy E-Receipt ≤50,000

### 3.3 อัตราภาษีขั้นบันได (เงินได้สุทธิ)
0–150,000 ยกเว้น · 150,001–300,000 = 5% · 300,001–500,000 = 10% · 500,001–750,000 = 15% · 750,001–1,000,000 = 20% · 1,000,001–2,000,000 = 25% · 2,000,001–5,000,000 = 30% · 5,000,001+ = 35%

### 3.4 กฎพิเศษ
- **WHT เครดิต** — รวม `withholding` ของทุก IncomeItem (ดอกเบี้ย 15% / ปันผล 10% / ฟรีแลนซ์ 3% ฯลฯ) หักจาก `taxBeforeCredit` → `taxDue` (ติดลบ = ขอคืน)
- **วิธีภาษีขั้นต่ำ 0.5%** — ถ้าเงินได้พึงประเมินประเภท 40(2)–(8) รวม ≥ 120,000 → `minimumTax = เงินได้(ไม่รวม 40(1)) × 0.5%` แล้ว `taxBeforeCredit = max(progressive, minimumTax)` **เว้นแต่** `minimumTax ≤ 5,000` ให้ใช้ progressive
- **เกณฑ์ต้องยื่น** — 40(1) อย่างเดียว: >120,000 (โสด)/220,000 (สมรส); มีเงินได้ประเภทอื่น: >60,000 (โสด)/120,000 (สมรส)
- **ปันผล (`dividendMode`)** — `final` (ดีฟอลต์): ถือ WHT 10% เป็น final ไม่นำมารวม; `include`: นำมารวมเป็น 40(4) + ใช้เครดิตภาษีเงินปันผล (gross-up)

## 4. การเดาเงินได้จากธุรกรรม (`detect.ts`)
- **40(1) เงินเดือน** — รายการ `direction:in`, ไม่ใช่ transfer, หมวด income, ที่เป็นรอบ ~30 วัน หรือ counterparty มี "เงินเดือน/payroll/salary" → รวมทั้งปี
- **40(4) ดอกเบี้ย** — counterparty/rawDesc มี "ดอกเบี้ย/interest" → amount + ประมาณ WHT 15%
- **40(4) ปันผล** — มี "ปันผล/dividend" → amount + ประมาณ WHT 10%
- เงินได้อื่น (40(2),(5)–(8)) — ระบบเดาไม่ได้แน่ → ให้ผู้ใช้เพิ่มเอง
- ทุกค่าที่เดา `source:'detected'` ผู้ใช้ override เป็น `'user'` ได้; **annualize** ×(12/เดือนข้อมูล) เมื่อ `annualize=true` และข้อมูล < 12 เดือน

## 5. คำแนะนำภาษี & แชท

- **`suggestSavings()` (pure-fn)** — คำนวณ "ช่องลดหย่อนที่เหลือ" ของแต่ละตัว (SSF/RMF/ThaiESG/ประกัน/บำนาญ/บริจาค/ดอกเบี้ยบ้าน) ตามรายได้+ที่ใช้แล้ว × **marginal rate** → `{label, used, cap, room, estimatedSaving, ref}` เรียงตามผลประหยัด; เคารพเพดานรวม 500k / 10% บริจาค / %รายได้
- **`filingInfo()` (rule-based)** — ต้องยื่นไหม → ฟอร์ม (ภ.ง.ด.90 ถ้ามีเงินได้นอก 40(1) ไม่งั้น ภ.ง.ด.91) → กำหนดยื่น (กระดาษ 31 มี.ค. / ออนไลน์ ~8 เม.ย. ปีถัดไป) → WHT ที่ขอคืนได้
- **แชท** — เสริม `server/src/services/chat.ts` จับคำถามภาษี (ภาษี/ลดหย่อน/ยื่น/SSF/RMF…) → ฉีดข้อเท็จจริงภาษีที่คำนวณแล้ว (เงินได้สุทธิ, ภาษี, marginal rate, top suggestions, กำหนดยื่น) ให้ AI เรียบเรียง + rule-based fallback
- **`taxAdvice()` ใน `gemini.ts`** — ผ่าน `generate()` (Groq→Gemini→rule-based) ตัวเลขล็อกด้วยโค้ด
- **ข้อควรระวัง** — แบนเนอร์ disclaimer ทุกที่

## 6. UI (`Tax.tsx` + เมนู 🧾 ภาษี)
1. แบนเนอร์ข้อควรระวัง (บนสุด)
2. การ์ดเงินได้แยกประเภท — ค่าที่เดา (แก้ได้) + เพิ่มประเภทอื่น
3. การ์ดค่าลดหย่อน — ฟอร์มจัดกลุ่มพับได้ (ครอบครัว / ประกัน+ปกส. / กองทุน / อื่นๆ) โชว์เพดาน + เหลือใช้สด
4. แผงผลลัพธ์ — เงินได้สุทธิ → ภาษีขั้นบันได (แยกขั้น) → เทียบวิธี 0.5% → หัก WHT → **ต้องจ่าย/ขอคืน** + แผงอธิบายสูตร+มาตรา
5. ตัววางแผนประหยัดภาษี — รายการคำแนะนำเรียงตามผลประหยัด + **สไลเดอร์สด** (เลื่อน SSF/RMF → ภาษีเปลี่ยนทันทีในเครื่อง แบบ `WhatIfSimulator`)
6. การ์ดเตือนยื่น — ต้องยื่นไหม/ฟอร์ม/นับถอยหลังกำหนด
7. หมายเหตุ annualize — สวิตช์ "ประมาณการทั้งปีจาก X เดือน" vs "ใช้ยอดจริงในช่วง" (ดีฟอลต์ = ประมาณการทั้งปี ติดป้ายชัด)
8. การ์ดเล็ก VAT ที่จ่ายในฐานะผู้บริโภค (7/107 ของยอดซื้อในช่วง) — เชิงข้อมูล

บันทึก TaxProfile → `PUT /api/tax` → refetch (ใช้ `version` ของ dataScope หรือ refetch ตรง)
